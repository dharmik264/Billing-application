# Phase 6: Generate Dynamic UPI QR Code on Bill - Execution Plan

## Goal
Implement a dynamic UPI QR code generator that takes the Shop's UPI ID and the Bill's Total Amount, and renders/prints a UPI URI QR code on the receipt.

## Wave 1: Backend Updates (Shop Model)

- [ ] **6-01-01: Update Shop model with UPI ID**
  - **File:** `backend/shop/models.py`
  - **Action:** Add a new `upi_id` `CharField` (e.g., max_length=50, null=True, blank=True) to the `Shop` model. This will store the merchant's UPI VPA (e.g., `phone@upi`).
  - **Automated Verify:** `python manage.py makemigrations --dry-run` shows changes.

- [ ] **6-01-02: Make migrations and migrate**
  - **Action:** Run `python manage.py makemigrations shop` and `python manage.py migrate` to apply the `upi_id` to the local SQLite database.
  - **Automated Verify:** Check `sqlite3 db.sqlite3 ".schema shop_shop"` contains `upi_id`.

- [ ] **6-01-03: Update Shop Serializer**
  - **File:** `backend/shop/serializers.py`
  - **Action:** Ensure `upi_id` is included in the fields list so it is returned to and writable from the Flutter frontend.
  - **Automated Verify:** Ensure tests pass or run a local GET on `/api/shop/` and verify `upi_id` is present.

## Wave 2: Frontend Data Models & UI

- [ ] **6-02-01: Update Shop Model in Flutter**
  - **File:** `lib/models/shop.dart` (or wherever Shop model is defined)
  - **Action:** Add `upiId` property. Update `fromJson` and `toJson` methods to handle the new `upi_id` field from the API.

- [ ] **6-02-02: Update Shop Setup UI**
  - **File:** `lib/screens/shop_setup_screen.dart`
  - **Action:** Add a `TextFormField` for entering the UPI ID. Validate it loosely (e.g., must contain `@`). Include it in the data map sent to the `saveShop` API call.

## Wave 3: Bill Printing / PDF Generation

- [ ] **6-03-01: Implement Dynamic QR Code in PDF / Receipt logic**
  - **File:** `lib/services/pdf_receipt_service.dart` (and thermal printing service if separate)
  - **Action:** Before adding the QR code to the layout, check if `shop.upiId` is set.
    - If set, construct the UPI URI: `upi://pay?pa=${shop.upiId}&pn=${shop.name}&am=${token.totalAmount}&cu=INR`
    - Use the receipt package's QR code widget/command (e.g., `BarcodeWidget` for PDF, or `.qrcode(uri)` for ESC/POS) to render the URI as a QR code at the bottom of the bill.
    - If not set, fallback to the existing static QR code logic or skip QR generation.

## Wave 4: End-to-End Verification

- [ ] **6-04-01: Verify UAT manually**
  - **Action:** Start the backend and frontend. Set a UPI ID in the shop setup. Create a token/bill. Generate the receipt PDF/Print preview. Use a mobile app (Google Pay / PhonePe / Paytm) to scan the generated QR code on the screen and verify that the payment amount is locked and pre-filled.
