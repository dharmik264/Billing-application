# REQUIREMENTS.md

This document serves as the master list of product and technical requirements for the Billing Application.

## 1. Functional Requirements

### 1.1 Offline Capabilities
- The system MUST be able to perform critical functions (Login, Shop Setup, Token Generation) even if the backend is completely unreachable (0-second timeout/bypass).
- Offline-generated tokens must be cached locally in SQLite or SharedPreferences.
- A background or manual sync task MUST be able to push cached tokens to the Django backend once a stable local Wi-Fi connection is re-established.

### 1.2 Performance & Rendering
- Lists of tokens, items, and receipts MUST be rendered using `ListView.builder` or `ListView.separated` to achieve memory efficiency and constant 60FPS scrolling.
- `SingleChildScrollView` must NOT be used for dynamically sizing lists containing more than 20 items.

### 1.3 Local Backend Connectivity
- The app must point to the specific local IP address of the machine running the Django server (e.g. `192.168.31.156`).
- It must handle `SocketException` and `TimeoutException` gracefully by displaying specific, actionable UI warnings (e.g. "Saved in Offline Mode (Backend Unreachable)") rather than halting the user flow.

## 2. Non-Functional Requirements

### 2.1 UX / UI
- Operations should feel instantaneous. If an API request is destined to fail due to network configurations, it should fail fast (0-3 seconds).
- Error messages must be non-blocking. Use Snackbars with appropriate color coding (e.g., Orange for Offline/Warning, Green for Success).

### 2.2 Security
- Network configurations should not rely on completely opening external firewalls; the setup must remain secure within the Private local network domain.
- Cleartext HTTP traffic must be enabled in `AndroidManifest.xml` since the backend lacks SSL certificates on the local network.
